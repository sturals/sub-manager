# SubManager

Self-hosted менеджер прокси-подписок: скачивает подписки (vless / vmess / shadowsocks / trojan, plain-списки, base64 и Clash YAML), убирает дубликаты, массово проверяет узлы через [Xray-core](https://github.com/XTLS/Xray-core) реальным HTTPS-запросом и отдаёт живые узлы одной подпиской для v2rayN, NekoBox и других клиентов.

## Возможности

- **Импорт:** списки ссылок (plain / base64), Clash YAML; протоколы `vless://`, `vmess://`, `ss://` (SIP002 и легаси base64), `trojan://`
- **Проверка:** один процесс Xray на чанк из 50 узлов (каждому узлу свой socks-inbound), узел считается живым только если через него реально прошёл запрос к `cloudflare.com/cdn-cgi/trace`
- **Дедупликация:** по конфигурации узла (md5 от outbound) и по реальному выходному IP
- **Умные ретесты:** мёртвые узлы перепроверяются с backoff (1 → 3 → 7 дней) и списываются после 5 провалов подряд; узлы, пропавшие из подписок, удаляются из базы
- **Метки:** живым узлам в remark проставляется флаг страны и реальный IP (`🇩🇪 DE - 1.2.3.4`)
- **Экспорт:** `GET /sub?token=...` — base64-подписка из живых узлов

## Установка (Windows)

```bat
git clone https://github.com/revishdubovzi23z/subvpn
cd subvpn
run.bat
```

`run.bat` сам поставит зависимости (`npm install`) и скачает Xray-core (`tools\get-xray.ps1`), затем запустит сервер на `http://localhost:3000`.

Вручную:

```bash
npm install
powershell -ExecutionPolicy Bypass -File tools/get-xray.ps1   # скачивает xray.exe + geo-данные
npm start
```

На Linux/macOS положите бинарник `xray` (без `.exe`) в папку `xray/` вместе с `geoip.dat`/`geosite.dat`.

## Использование

1. Откройте `http://localhost:3000`, добавьте URL подписок.
2. Нажмите **Запуск** — узлы скачаются, отфильтруются и проверятся (живые логи и прогресс на странице).
3. Скопируйте URL экспорта и вставьте его в клиент (v2rayN → подписки, NekoBox и т.д.).

## Доступ с других устройств и безопасность

- Запросы **с этого же компьютера** работают без ограничений.
- Запросы **с других устройств** (телефон в той же сети) требуют токен: URL экспорта на дашборде уже содержит `?token=...`. Токен генерируется автоматически и хранится в `database.json` (`settings.token`).
- Запросы с доменным `Host`-заголовком отклоняются — защита от DNS-rebinding.
- Хост/порт настраиваются переменными окружения: `HOST` (по умолчанию `0.0.0.0`), `PORT` (по умолчанию `3000`). Для строго локального режима: `set HOST=127.0.0.1`.

## API

| Метод | Путь | Описание |
|---|---|---|
| GET | `/api/subs` | список подписок |
| POST | `/api/subs` | сохранить список `{ "urls": [...] }` |
| POST | `/api/run` | запустить полный цикл |
| POST | `/api/stop` | остановить проверку (без порчи данных) |
| GET | `/api/status` | прогресс + последние логи |
| GET | `/api/stats` | счётчики узлов |
| GET | `/sub` | base64-подписка из живых узлов |

## Данные

- `database.json` — подписки, узлы, токен. Держится в памяти, на диск пишется атомарно с дебаунсом (2 с). **В git не коммитится.**
- `app.log` — лог, ротация при превышении 1 МБ.
- `xray/config.json` — генерируется при каждой проверке.
